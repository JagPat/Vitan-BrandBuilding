# Kaizen — Continuous Improvement Protocol for Vitan Growth OS

## Purpose
Kaizen means "change for better." This protocol ensures the Growth OS is self-evolving.

## The Three Kaizen Loops

### Loop 1: Per-Issue Retrospective (every completed issue)
After marking any issue "done", add retrospective comment:
1. What worked 2. What didn't work 3. What I'd do differently 4. System improvement suggestion

### Loop 2: Weekly Growth OS Review (every Friday)
PA creates weekly-review-{date}.md with: pipeline snapshot, intelligence quality, outreach effectiveness, agent performance, improvements implemented/proposed, capability gaps.

### Loop 3: Monthly Strategic Retro (first Monday of month)
PA creates monthly-retro-{YYYY-MM}.md with: growth metrics, competitive changes, system evolution, what works vs what needs redesign, strategic pivots, agent upgrades proposed, board input needed.

## Self-Evolution Rules
- Never repeat a known mistake from retrospectives
- Document learnings at shared-workspace/intelligence/learnings/
- Propose capability updates via KAIZEN: issues
- Track improvement velocity in monthly retros

## Agent-Specific
- BB: track outreach effectiveness, build outreach-playbook.md
- PA: assess positioning quality, evaluate cycle cadence
- HR: track agent health trends, identify recurring gaps
- FE: maintain tech-debt.md, track tool utilization

## Kaizen Issues: "KAIZEN: {desc}" at medium priority
## Anti-Patterns: retro without action, improvement sprawl, ignoring board feedback
## Success: every issue has retro, weekly reviews consistent, response rates improve
