## Objective
Increase new-client pipeline reliability by combining local trust-surface correction with tightly measured outbound touches in a single weekly operating loop.

## Why This Matters Now
- Competitor scale gaps are large; Vitan can win faster on trust clarity + response quality.
- Weekly publishing alone is not enough unless discovery surfaces and outbound execution are synchronized.
- Week-16 is the right cycle to operationalize conversion-first discipline before next send windows.

## Execution Block (Week-16)

1. Local trust-surface sprint (Top 10)
- Validate and correct canonical identity across the highest-impact listings first.
- Canonical fields:
  - Name: `Vitan Architects`
  - Website: `https://www.vitan.in`
  - Contact: `connect@vitan.in`
- Priority surfaces:
  - Google Business Profile
  - JustDial
  - IndiaMART
  - key architecture/company aggregators already identified in VITA-134

2. Outbound execution lane (First 5 accounts)
- For each approved account touch, enforce this sequence:
  - proof-led opener (project relevance)
  - one friction insight
  - one concrete CTA (20-minute diagnostic)
  - follow-up with one visual proof and expected outcome
- SLA target: same-day acknowledgment and 24h first substantive response.

3. Measurement lane
- Track both lanes in a single weekly review pack:
  - trust corrections completed / pending / blocked
  - outbound touches / replies / meetings / proposals started
  - response latency distribution

## Weekly Targets (Week-16)
- Trust lane: 5+ priority listing corrections completed.
- Outbound lane: 5 approved outbound touches executed.
- Conversion lane: >=2 warm replies, >=1 meeting booked.

## Risk Controls
- No external send without explicit approval gate.
- If directory edit requires ownership verification and blocks progress, log blocker and move to next listing.
- Avoid one-off metric swings; evaluate trend direction over repeated cycles.
